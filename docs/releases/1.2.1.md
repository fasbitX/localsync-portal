# Release 1.2.1

**Date:** 2026-04-15
**Type:** Patch - Bug Fix

## Issue

Remote folder tree did not match the local client's parent-child hierarchy. When the local filesystem had nested folders like `2024/Football/Game1`, the remote only stored the leaf folder (`2024/Football/Game1`). Parent folders (`2024`, `2024/Football`) were never created in the remote database.

This caused the startup reconciliation to repeatedly detect parent folders as "missing from remote" and attempt to create them on every cycle, producing unnecessary API calls and empty folder records.

## Root Cause

`UploadController.findOrCreateFolder()` and `POST /api/folders` only created the target folder -- they did not walk the path segments to ensure ancestor folders existed in the database.

## Fix

Added `ensureAncestorFolders(String folderPath)` to `UploadController`. This method splits the path on `/`, iterates over all segments except the last (leaf), and creates a DB record + disk directory for each ancestor that doesn't already exist.

Wired into both folder creation entry points:
- `findOrCreateFolder()` -- called during photo upload (`POST /api/upload`)
- `POST /api/folders` -- called during explicit folder creation from local sync

## Files Changed

### Remote Server
- **`UploadController.java`** -- added `ensureAncestorFolders()`, called from `findOrCreateFolder()` and `createFolder()`
