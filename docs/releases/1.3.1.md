# Release 1.3.1

**Date:** 2026-04-15
**Type:** Patch - Bug Fix

## Issue

Photos on the remote server (gallery and admin) were displayed out of chronological order. The local client sorts photos alphabetically by filename (which is chronological for camera-named files like IMG_0001, IMG_0002), but the remote used different ordering.

## Root Cause

- **Gallery endpoint** (`findAllByFolderIdAndVisibleTrue`) had no ORDER BY clause -- photos returned in arbitrary database insertion order
- **Admin and sync endpoints** (`findAllByFolderIdOrderByUploadedAtDesc`) ordered by upload timestamp descending -- this reflects when the file was synced to the server, not the photo's actual order. Photos uploaded out of order (retries, queue batching) appeared scrambled.

## Fix

Changed all photo queries to order by `filename ASC`, matching the local client's `files.sort((a, b) => a.name.localeCompare(b.name))`.

- `findAllByFolderIdAndVisibleTrue` -> `findAllByFolderIdAndVisibleTrueOrderByFilenameAsc`
- `findAllByFolderIdOrderByUploadedAtDesc` -> `findAllByFolderIdOrderByFilenameAsc`

## Files Changed

### Remote Server
- **`PhotoFileRepository.java`** -- renamed query methods to sort by filename ascending
- **`GalleryController.java`** -- updated to use new query method
- **`SyncController.java`** -- updated to use new query method
- **`AdminController.java`** -- updated to use new query method
