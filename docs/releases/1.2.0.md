# Release 1.2.0

**Date:** 2026-04-15
**Type:** Minor - New Feature

## Summary

The remote server is now a display-only mirror. On startup, the local client reconciles with the remote to force an exact match. All admin modification capabilities have been removed from the remote server.

## New Feature: Startup Reconciliation

When the local client starts, it runs a full reconciliation against the remote server:

1. Fetches the complete remote state via `GET /api/sync/state`
2. Scans the local filesystem for all folders and image files
3. Deletes remote folders that don't exist locally
4. Deletes remote photos that don't exist locally
5. Creates remote folders that exist locally but not on remote
6. Queues local photos missing from remote for upload

This ensures stale data from before sync propagation was added gets cleaned up, and any drift between local and remote is corrected on every restart.

## Change: Remote Admin Panel is Now Display-Only

### Removed from AdminController
- `DELETE /api/admin/folders/{id}` (folder deletion)
- `DELETE /api/admin/photos/{id}` (photo deletion)
- `PATCH /api/admin/folders/{id}/visibility` (visibility toggle)
- `PATCH /api/admin/photos/{id}/visibility` (visibility toggle)

### Removed from Admin Dashboard UI
- All delete buttons (folder and photo)
- All visibility toggle buttons
- Confirmation dialog overlay
- All associated JavaScript handlers

### Kept (read-only)
- `GET /api/admin/folders` -- view folder list
- `GET /api/admin/folders/{id}/photos` -- view photos in a folder
- Admin login/logout

## Files Changed

### Remote Server
- **`SyncController.java`** -- added `GET /api/sync/state` endpoint
- **`AdminController.java`** -- stripped to read-only (4 methods removed)
- **`SecurityConfig.java`** -- added `/api/sync/**` to auth matchers
- **`admin.html`** -- removed action buttons and confirm dialog
- **`admin.js`** -- removed all modification handlers and UI code

### Local Client
- **`sync.js`** -- added `scanLocalState()` and `reconcile()` functions
- **`index.js`** -- calls `sync.reconcile()` on startup
