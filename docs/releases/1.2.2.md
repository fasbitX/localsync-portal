# Release 1.2.2

**Date:** 2026-04-15
**Type:** Patch - Bug Fixes

## Issue #1: Admin folder sidebar displayed as flat list

The admin dashboard rendered all folders as a flat column with no visual hierarchy. Parent folders like `2024` appeared at the same indentation level as deeply nested children like `2024/Football/Game1`.

### Root Cause

`renderFolderList()` in `admin.js` iterated the flat folder array and rendered each item identically with `displayName` only -- no tree building, no depth awareness.

### Fix

- Added `buildFolderTree()` to `admin.js` -- converts the flat folder array into a parent-child tree using `relativePath` segments (same algorithm as the local client's `buildFolderTree()`)
- `renderFolderList()` now renders the tree recursively with `paddingLeft` indentation based on depth (20px per level)
- Intermediate parent nodes that exist only as path segments (no DB record) render as non-clickable, dimmed labels
- Children sorted alphabetically at each level
- Changed `GET /api/admin/folders` to sort by `relativePath` ascending (was `createdAt` descending)
- Added `findAllByOrderByRelativePathAsc()` to `FolderRepository`

## Issue #2: Gallery links returned literal "forward:/gallery.html" text

Sharing a gallery URL like `/gallery/{uuid}` displayed the raw string `forward:/gallery.html` instead of the gallery page.

### Root Cause

`GalleryController` was annotated with `@RestController`. With `@RestController`, all methods are implicitly `@ResponseBody`, so `return "forward:/gallery.html"` was serialized as a plain text response body instead of being processed as a Spring MVC server-side forward directive.

### Fix

- Changed `@RestController` to `@Controller` on `GalleryController`
- Added `@ResponseBody` to the four API/resource-serving methods (`getFolderInfo`, `getGalleryPhotos`, `servePhoto`, `downloadPhoto`) so they continue to return JSON/binary responses
- The `galleryPage()` method (no `@ResponseBody`) now correctly performs a server-side forward to `gallery.html`
