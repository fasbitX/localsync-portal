# Release 1.4.1

**Date:** 2026-04-15
**Type:** Patch - Bug Fix

## Issue

When dragging a new photo onto a folder, all thumbnails in the detail panel disappeared. The photo grid went blank and the user had to re-select the folder to see photos again.

## Root Cause

Both upload paths (drag-and-drop and modal) called `renderFolders()` after a successful upload. This function rebuilds the **entire page** -- folder tree and detail panel -- resetting the detail panel to the empty "Select a folder to view photos" state. The selected folder context and its photo grid were destroyed.

## Fix

Added `refreshAfterUpload(folderPath)` which does two things without rebuilding the page:

1. **Updates folder tree counts in-place** -- fetches fresh folder data, walks existing `.tree-node` elements, and updates the `.tree-meta` count spans. No DOM rebuild, no loss of expand/collapse state.
2. **Reloads the photo grid** -- if the currently viewed folder matches the upload target, calls `loadFolderPhotos()` to refresh thumbnails with the new photo in its proper chronological (alphabetical) position.

Replaced `renderFolders()` calls in both the drag-and-drop `xhr.onload` handler and the modal upload `xhr.onload` handler.

## Files Changed

### Local Client
- **`ui/app.js`** -- added `refreshAfterUpload()`, replaced `renderFolders()` in both upload success handlers
