# Release 1.4.0

**Date:** 2026-04-15
**Type:** Minor - New Feature

## New Feature: Zip File Upload Support

Users can now drag-and-drop or select `.zip` files as a method of bulk-loading photos into a folder. The server extracts image files from the zip, saves them to the target folder, and the watcher syncs them automatically.

### How It Works

1. Drop a `.zip` file (or mix of images and zips) onto a folder in the tree, or select via the upload modal
2. The server receives the zip, extracts all image files (JPG, PNG, TIFF, RAW, CR2, NEF, ARW)
3. Non-image files and directories inside the zip are skipped
4. Duplicate filenames are auto-renamed using the `filename (1).ext` pattern
5. The zip file is deleted after extraction
6. The watcher picks up the extracted images and syncs them to the remote

### Details

- Zip files bypass the client-side duplicate pre-check (since contents can't be read in the browser) -- duplicates are handled server-side during extraction
- Regular image files in the same upload still go through the normal duplicate check dialog
- Nested directories inside zips are flattened -- only the filename is used, not the zip's internal path structure
- Hidden files (starting with `.`) inside zips are skipped

### Dependencies

- Added `adm-zip` package for zip extraction
