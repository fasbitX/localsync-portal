# Release 1.5.1

**Date:** 2026-04-15
**Type:** Patch - Bug Fix

## Fix: Zip File Drag-and-Drop Upload Failing

Dropping a `.zip` file (or any file) outside a folder tree row caused the browser to navigate away or attempt to download the file, making zip uploads appear broken.

### Changes

- **Global drop prevention** -- added document-level `dragover`/`drop` handlers to stop the browser from opening or downloading dropped files
- **Detail panel drop zone** -- the photo detail panel (right side) now accepts file drops when a folder is selected, with a dashed-outline visual highlight
- **Better error feedback** -- upload failures now show an `alert()` with the error message instead of a brief 3-second text flash on the tree label
- **Empty zip handling** -- if a `.zip` upload extracts zero photos, a clear message explains the zip must contain image files
- **Updated help text** -- upload modal and empty-state text now mention ZIP as a supported format

### Files Changed

- **`local-client/src/ui/app.js`** -- extracted `handleFileDrop()` function, added detail panel drop listeners, global drop prevention, improved error alerts, updated copy
- **`local-client/src/ui/style.css`** -- added `.detail-dragover .card` style for drop highlight
