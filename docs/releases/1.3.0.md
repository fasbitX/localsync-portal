# Release 1.3.0

**Date:** 2026-04-15
**Type:** Minor - New Feature

## New Feature: Duplicate File Detection on Upload

When uploading photos via drag-and-drop or the upload modal, the local client now checks for existing files with the same name in the target folder before saving.

### Behavior

1. Before upload, filenames are checked against the target folder on disk
2. If no duplicates, upload proceeds normally
3. If duplicates found, a dialog appears listing each conflicting file with two options:
   - **Skip** -- file is not uploaded (default)
   - **Rename** -- file is saved with an auto-generated unique name (e.g. `photo (1).jpg`)
4. After resolving all conflicts, upload proceeds with the remaining/renamed files

### How Renaming Works

- Appends ` (N)` before the extension: `IMG_001.jpg` -> `IMG_001 (1).jpg`
- Increments the counter until a unique name is found on disk
- The renamed file is uploaded with the new name, so the watcher syncs it under the new name

### API

New endpoint: `POST /api/folders/:folderPath/check-duplicates`
- Request body: `{ "filenames": ["photo1.jpg", "photo2.jpg"] }`
- Response: `{ "duplicates": ["photo1.jpg"], "suggestions": { "photo1.jpg": "photo1 (1).jpg" } }`

## Files Changed

### Local Client
- **`routes/folders.js`** -- added `POST /api/folders/:folderPath/check-duplicates` endpoint
- **`ui/app.js`** -- added `checkDuplicates()`, `showDuplicateDialog()`, `checkAndResolve()` helpers; updated both drag-and-drop and modal upload handlers to check for duplicates before uploading
