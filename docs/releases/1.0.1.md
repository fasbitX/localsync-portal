# Release 1.0.1

**Date:** 2026-04-15
**Type:** Patch - Bug fixes and UI improvements

## Issues Fixed

### Remote upload fails with HTTP 500 on retries
- **Issue:** When a photo upload to the remote server was retried (after a timeout or network error), the remote server returned HTTP 500. All 3 retry attempts would fail, leaving files permanently stuck in "failed" status in the sync log.
- **Root cause:** The `photo_files` table has a `UNIQUE` constraint on `relative_path`. On the first attempt, the file was stored on disk and the DB record inserted. If the response didn't reach the local client (timeout, network blip), the retry would try to INSERT a duplicate record, throwing an unhandled `DataIntegrityViolationException`.
- **Fix:** Upload handler now checks for an existing photo record before inserting. If found, it updates the existing record instead of inserting a duplicate. (`UploadController.java`, `PhotoFileRepository.java`)

### Folder appearance changes when photos are added
- **Issue:** Adding photos to a folder changed its icon from a folder to a camera, changed the text style from uppercase/muted to normal, and generally altered the look and feel of the tree -- making it confusing.
- **Fix:** All folders now display consistently with the same icon and text style regardless of whether they contain photos. Image count metadata still displays.

### Folder names truncated with ellipsis
- **Issue:** Long folder names (e.g. "Scrimmage - April 11th-2026") were cut off with "..." in the folder tree panel because of `white-space: nowrap` and `text-overflow: ellipsis`.
- **Fix:** Folder labels now wrap instead of truncating.

## Changes

### Photo viewer with thumbnail grid and fullscreen overlay
- Folder contents now display as a **thumbnail grid** (the original view was lost in a prior update)
- Clicking a thumbnail opens a **fullscreen overlay viewer** with:
  - Large image maximized on screen while maintaining aspect ratio
  - `<` and `>` buttons to navigate between all images in the folder
  - Left/right arrow keyboard navigation
  - Image filename and EXIF metadata displayed at the bottom
  - Close via X button, ESC key, or clicking the backdrop

### Individual photo deletion
- **New endpoint:** `DELETE /api/folders/:folderPath/photos/:photoName` on the local client
- Delete button in the fullscreen photo viewer
- Confirmation dialog warns that deletion does not remove the photo from the remote server
- Cleans up the corresponding `sync_log` entry

### Photo serving for thumbnails
- Local client now serves photos from the watch directory via `/photos/` static route for thumbnail display in the UI
