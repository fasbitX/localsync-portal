# Release 1.5.0

**Date:** 2026-04-15
**Type:** Minor - New Feature

## New Feature: Folder Details (Game Info & Notes)

Folders can now store structured game details that display on the public gallery page.

### Fields

- **Opponent** -- team name (e.g. "Milton Eagles")
- **Location** -- venue (e.g. "Home Field")
- **Game Date & Time** -- datetime picker
- **Final Score** -- free text (e.g. "7-3")
- **Notes** -- large text area for game recap, highlights, etc.

### Access Points (Local Client)

1. **Right-click context menu** -- "Details" option on any folder
2. **Details button** -- in the folder detail panel header, next to the photo count

Both open a modal form pre-populated with existing data. On save, details are stored locally and synced to the remote server.

### Gallery Display (Remote Server)

Details appear above the photo grid on the public gallery page:
- Meta line showing opponent, location, date, and score separated by dots
- Notes section below with preserved line breaks

### Database Changes

**Local** (`local-schema.sql`): New `folder_details` table with `relative_path` (unique), `location`, `game_date`, `score`, `opponent`, `notes`

**Remote** (`remote-schema.sql`): Added `location`, `game_date`, `score`, `opponent`, `notes` columns to `folders` table

### API Changes

**Local Client:**
- `GET /api/folders/:folderPath/details` -- fetch details
- `PUT /api/folders/:folderPath/details` -- save details (upserts locally, syncs to remote)

**Remote Server:**
- `PUT /api/folders/details?relativePath=...` -- receive details from local sync (API key auth)
- `GET /api/gallery/{uuid}/info` -- now includes detail fields in response
